You are a senior backend + mobile engineer working on a production-ready AI photo tagging system.


The system consists of:

* Django REST backend (photo processing, tagging, face grouping)
* FastAPI LLaVA service (image captioning)
* Android client (Kotlin, Room, Retrofit)
* Docker-based deployment

Your task is to **improve the project to a near-production level WITHOUT rewriting it from scratch**.

---

## 🎯 Goals

Improve architecture, API design, and reliability while preserving existing functionality.

---

## 🧩 Backend Improvements (Django)

1. **Refactor API design**

   * Split `/api/upload/` into:

     * POST `/api/photos/upload/`
     * GET `/api/photos/`
     * GET `/api/albums/`
     * GET `/api/faces/`
   * Add serializers instead of manual dict building
   * Use DRF ViewSets where appropriate

2. **Add pagination and filtering**

   * Pagination for photos
   * Filter by:

     * tags
     * face_number
     * category

3. **Add basic authentication layer**

   * Replace `device_id` with:

     * either token-based auth OR
     * keep device_id but validate format and uniqueness
   * Prepare structure for future user accounts

4. **Improve error handling**

   * Standardize API responses:
     {
     "status": "error",
     "message": "...",
     "details": ...
     }
   * Catch and log LLaVA / face detection failures cleanly

5. **Optimize face matching**

   * Move cosine similarity logic into a separate service class
   * Add caching or precomputed embeddings if possible

6. **Code cleanup**

   * Remove unused files (e.g. temporary scripts)
   * Split large files like `views.py` into modules:

     * services/
     * selectors/
     * serializers/
